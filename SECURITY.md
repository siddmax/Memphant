# Security Policy

Please report security issues privately through GitHub security advisories for this repository.

Do not file public issues, examples, or docs containing secrets, exploit details, private benchmark fixtures, customer data, production telemetry, live project refs, credentials, or account-operation traces.

MemPhant treats memory poisoning, cross-tenant leakage, deletion incompleteness, citation forgery, selector injection, and exposed service keys as release-blocking security issues.
