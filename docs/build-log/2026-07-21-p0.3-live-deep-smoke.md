**Goal:** Prove the REAL Azure provider (not the mock) emits valid tool calls and
Deep completes ONE valid live pair end-to-end through the actual recall pipeline,

(`deep_recall.rs:948`) and the resolved P0.4 ingestion gate.

`memphant-eval run benchmarks/rung12-l4-exhaustive-sampled.yaml --l4-runtime-provider`
with the campaign's `sonnet` Deep env. This runs the ONE buried-evidence case
(`l4_exhaustive_raw_episode_buried`: query "What is required before Stargate

units merged as `RecallChannel::Deep` → packing → trace with `deep` summary +
`cost_micros`. The `$0.30`/recall cap is enforced by the provider itself
(`deep_recall_openrouter.rs:31`, 300_000 micros). InMemory store: the only
pipeline stage not exercised here is the Postgres `fetch_deep_snapshot`, covered
separately by the resolved P0.4 ingest and by A2.

Free local-stub prevalidation (no spend): the identical command WITHOUT

The smoke PASSES iff ALL hold:

1. **Case passes** (`eval=pass 1/1`): the buried unit
   `mem_stargate_raw_episode_answer` is in the Deep `candidate_whitelist`, cited,
   packed at position 1, the decoy `mem_stargate_decoy` is forbidden-absent, and

4. **Settlement receipt present**: a `deep_settlement` line is emitted with

   Recorded EVEN ON ABORT (a Capped/Partial trace still settles and prints).

Kill-switch: if P0.3 fails, paid lanes stay closed; capture the diagnostic and
do NOT retry blindly (systematic-debugging).

## Result — DONE_WITH_CONCERNS (plumbing PROVEN; a real tool defect found + fixed; case-quality not proven)

**Verdict against the frozen predicate:** predicates 2 (≥1 tool iteration), 3
(≤$0.30), 4 (settlement receipt) all PASS on every run. Predicate 1 (the buried
case passes with the gold answer) does NOT pass. Net: the **live-provider
plumbing claim P0.3 exists to gate is proven**; the case-level *quality*
assertion is not — which is A2/A3 territory and, per A1, moot for the dev misses.

**Total live spend:** $0.1288 across 10 runs (5 diagnostic + 1 instrumented + 4
confirmation); max single recall $0.0176, every run ≤ the $0.30/recall cap.

### What the smoke found (systematic-debugging, not blind retry)

Runs 1–5 (pre-fix): 1× `Completed` (but nominated no gold), 4× `Partial /
InvalidOutput` with **empty** gathered evidence. Reproducible, not noise. An
instrumented run (`MEMPHANT_DEEP_DIAG`, since removed) captured the exact
trajectory:

```
search_files("Stargate deploy")  -> BOTH sources found (decoy + heliotrope answer)
read_file(episode, start=1, end=50) -> {"error":"invalid_range"}
read_file(other,   start=1, end=50) -> {"error":"invalid_range"}
=> 2 malformed responses > malformed_response_limit(1) => Partial/InvalidOutput, empty
```

**Root cause: a real `read_file` tool defect, not a model failure.** Episode
bodies are 1 line; the model idiomatically reads a generous range (`end_line:
50`) to see the whole file; `read_file` rejected any `end_line > lines.len()` as
`invalid_range` (`deep_recall_openrouter.rs:1396`) instead of clamping like
`head -N`. Two such rejections trip `malformed_response_limit=1` and abort Deep
with empty evidence. This is why **Deep had never produced one valid live pair**
(standing STATUS rung-12 gap): every real Deep recall over short episode bodies
hit this. The single-turn `f32fdb37` stream diagnostic never exercised a later
`read_file` iteration, so it read `production_parser_first_failure: null` while
this defect sat one iteration deeper.

**Fix (TDD):** clamp `end` to the file length; only `start` past EOF errors
(`read_file_clamps_end_line_past_eof_instead_of_erroring`). Post-fix: **7/7 runs
`status=Completed, stop_reason=Completed`**, 5 tool iterations, 2 nominated
source UUIDs, ~$0.017 settled — the first reliable end-to-end live Deep pairs.

### Remaining concern (not a plumbing failure)

The buried case still fails: `missing_units=[mem_stargate_raw_episode_answer]`,
`forbidden_present=[mem_stargate_decoy]`. Deep now finds and nominates evidence,
but the decoy ("Stargate deploy requires a routine restart.") is a legitimate
surface answer to "What is required before Stargate deploy?", so Sonnet's
nomination surfaces the decoy unit rather than only the buried "heliotrope"
answer. That is a retrieval-*quality* outcome on an adversarial synthetic case,
gated by A2/A3 — not the live-plumbing claim P0.3 owns. And per **A1**, Deep (a
depth lever) cannot fix the dev Fast-misses regardless (0% absent-from-pool), so
the paid depth lane does not open on this evidence.
