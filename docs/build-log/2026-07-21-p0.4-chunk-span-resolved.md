**Task:** Phase 0 P0.4 of the tri-domain SOTA plan
(`docs/superpowers/plans/2026-07-21-tri-domain-sota-plan.md`). Prove the

## Verdict: RESOLVED (did not recur)

One full exposed ingestion of the pinned LME-V2 dev corpus reached **670/670
sources** through the packaged `MemoryService<PgStore>` runtime with
`MEMPHANT_RESOURCE_CHUNKS=on` (the exact failing path) and **zero LLM/model
calls**. The chunk-span conflict **did not recur**. No fix was required — the
diagnostic-capture commit `9e53d8c4` already proved both chunkers correct at
mint time, and this run confirms empirically that no re-compile/retry path
produces the divergence under real full-scale load.

## How it was run (all free, no model spend)

```
python3 scripts/run_lme_v2_p1_t6.py acquire     --directory <data>
python3 scripts/run_lme_v2_p1_t6.py materialize  --directory <data> --output <mat>
python3 scripts/run_lme_v2_p1_t6.py verify-no-model \
  --fixture exact --case-id 19367bc7 \
  --base-database-url postgres://memphant:memphant@localhost:5432/memphant \
  --directory <data> --materialized <mat> --output <out>
```

- **Dataset:** pinned LME-V2 corpus, sha256-locked
  (`benchmarks/manifests/longmemeval_v2.lock.json`). `acquire` verified every
  file hash; `trajectories.jsonl` = 1,195,604,539 bytes,
  sha256 `363cec9a…f3c6`. `materialize` and `acquire` both reported
  `paid_calls: 0`.
- **Ingestion:** the `exact` fixture inserts 500 trajectories (retained as
  670 resource-fragments) then drains the reflect queue. This is the construction
  (retain + reflect-compile) half only — no reader/deep rows.
- **Run-owned Postgres (P0.1 discipline):** `verify-no-model` runs under
  `scripts/with_scratch_db.sh`, which mints an ephemeral migrated database from
  the base URL and force-drops it afterward. The shared `memphant` DB is never
  touched. Source DB was `memphant_scratch_92696_1784683939`; cleanup verified
  `orphan_clone_count: 0`, `source_job_state {dead:0, pending:0, total:0}`,
  `force_drop_verified: true`.
