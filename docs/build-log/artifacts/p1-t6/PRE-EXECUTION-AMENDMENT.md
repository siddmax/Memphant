# P1-T6 pre-execution amendment

Before any reader, judge, or Deep output and before any paid request, review found that the preregistered selection digest `ffe151038e3dc54c8132b58a2d39575db9ee37d0ead8f873afda67a6e35c2bea` had no serialization definition and could not be reproduced from the frozen rows using ordinary canonical encodings.

The frozen IDs and answer-blind selection algorithm are unchanged. The replacement contract hashes UTF-8 canonical JSON produced with `json.dumps(rows, sort_keys=True, ensure_ascii=True, separators=(",", ":"))`; rows are sorted by `(domain, ability, id)` and contain exactly `domain`, `ability`, `question_type`, and `id`. Its digest is `d7762dbaffff7acfe779162d4993c8c09ef0440e3c1a25e0d3408127d73e25fa`.

The same pre-output review replaced floating candidate aliases with dated dispatch slugs, pinned the official reader to one DeepInfra BF16 route through a fail-closed loopback request policy, and replaced the floating official judge alias with `gpt-5.2-2025-12-11`. No benchmark output was observed and no billable call was made before this amendment.

Before any billable call, a fresh public endpoint check showed that additive Azure inventory metadata had changed while every material registered route predicate remained intact (reader inventory unchanged; Sonnet `dd421291...`, Luna `cac4282e...`, Sol `f38ef32f...`). The gate therefore archives full inventory and matching-entry hashes as observations but gates only the frozen material contract: exact dated endpoint name and model ID, provider/tag/quantization where applicable, required tool and token parameters, minimum context/completion limits, and prices no higher than the registered ceilings. This prevents harmless metadata ordering/additions from invalidating a restart without weakening model, provider, capability, privacy, or cost locks. No benchmark output was observed and no billable call was made before this refinement.

A final independent no-spend audit found three execution-proof omissions before any benchmark output existed. This amendment closes them by sending the reader's frozen `deepinfra` + `bf16` + per-million-token `max_price` policy in the paid request itself; independently fetching and archiving a sanitized OpenRouter receipt for every Deep generation ID and requiring exact ID, Azure route, dated model, prompt-token, and micro-USD agreement; and constructing coordinator, server, and official-harness environments from a narrow allowlist so ambient credentials and `MEMPHANT_DEEP_*` overrides cannot cross process boundaries. Each candidate's complete Rust runtime configuration hash is now frozen in the manifest, independently reproduced in Python, and bound to the pre-execution proof and every successful row. Missing or mismatched receipts consume the full row reservation and make the treatment ineligible without replay. API keys plus full, encoded, and decoded scratch-Postgres credential variants are redacted from nested artifacts in the server-cleanup `finally` path. No benchmark output was observed and no billable call was made before this amendment.
