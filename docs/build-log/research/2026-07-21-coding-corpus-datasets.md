# Deep research: public coding-agent trajectory corpora (2026-07-21)

**Question:** rebuild a coding-agent execution-trace corpus from zero (local events DB wiped) — for a code-profile retrieval corpus + an adversarial buried-evidence golden.

## Summary

As of mid-2026 there is abundant public, permissively-licensed material to rebuild a coding-agent execution-trace corpus from zero: at least seven downloadable HuggingFace datasets ship genuine multi-turn tool-use transcripts (assistant reasoning/actions + tool/environment observations, not (issue,patch) pairs) totaling well over 900K trajectories, all under CC-BY-4.0 or MIT — the largest being NVIDIA's SWE-Zero-openhands-trajectories (318,115), Open-SWE-Traces (207,489, 9 languages), nebius/SWE-agent-trajectories (80,036) and SWE-rebench-openhands-trajectories (67,074, avg 64.3 turns), plus SWE-Gym's rollout sets and the MIT-licensed SWE-smith-trajectories (5,017). For the "code-profile retrieval corpus" (a) these are directly ingestible with a schema adapter, since every one exposes an OpenAI-style role/content/tool_calls transcript that maps 1:1 to the assistant-message + tool-result span extraction the pipeline already does. For the adversarial buried-evidence golden (b), the strongest 2026 template is SWE-ContextBench, which builds a continuity corpus by recording full agent trajectories (avg 25,633.7 tokens each) as a retrievable context pool and derives "related tasks" via human-verified GitHub issue/PR dependency mining (not LLM span-QA) graded by FAIL_TO_PASS/PASS_TO_PASS tests; LongMemEval supplies the complementary needle-in-haystack recipe (insert evidence sessions among unrelated distractors from ShareGPT/UltraChat with plausible timestamps) that directly matches the pipeline's LLM span-located QA approach. Ranked by effort-to-value, the fastest path is: ingest nebius/SWE-rebench-openhands-trajectories or SWE-smith-trajectories first (clean schema, deep turns, permissive license), then layer NVIDIA's 200K-300K sets for volume, and adopt LongMemEval's insertion method for span-QA generation while borrowing SWE-ContextBench's dependency-mining idea for a higher-quality golden slice. The main caveat: nearly all high-volume sets are synthetic distillation rollouts (model-generated, not organic production logs), which is fine for retrieval-corpus and buried-evidence-golden purposes but is not "real human agent traffic."

## Findings (adversarially verified)

### [high] NVIDIA SWE-Zero-openhands-trajectories is the single largest ingestible corpus: 318,115 multi-turn OpenHands trajectories over 118,092 distinct issues, CC-BY-4.0, with each row a full system/user/assistant/tool conversation history (tool = environment observations) plus a final unified-diff patch — not (issue,patch) pairs.

Dataset card states verbatim 'Total Trajectories: 318,115' and 'Total Issues: 118,092'; the trajectory field is 'Complete conversation history with roles: system/user (initial prompts), assistant (model reasoning/actions) and tool (environment observations)'; license 'Creative Commons Attribution 4.0 International License (CC BY 4.0)'. Parquet, 100K-1M size class. Merges claims 19, 20, 21 (all 2-1 or 3-0, same primary source).

Sources: https://huggingface.co/datasets/nvidia/SWE-Zero-openhands-trajectories

### [high] NVIDIA Open-SWE-Traces provides 207,489 multi-turn agentic trajectories spanning nine programming languages (Python, Go, TS, JS, Rust, Java, PHP, C, C++), split ~105k OpenHands / ~102k SWE-agent, CC-BY-4.0, 18.3 GB — heterogeneous framework logs normalized into explicit role/content/tool_calls fields (~94 avg assistant turns/trajectory), directly ingestible as a multilingual code-profile retrieval corpus.

HF card: '207,489 total trajectories'; per-language breakdown sums to 207,489 (Python 48,179 / Go 46,834 / TS 36,897 / JS 29,360 / Rust 21,735 / Java 13,176 / PHP 10,323 / C 683 / C++ 302). arXiv 2606.16038 (NVIDIA, June 2026): '207,489 agentic trajectories spanning nine programming languages'; paper 'unified the heterogeneous interaction histories produced by the OpenHands and SWE-agent frameworks into a consistent format' with 'explicit role, content, and tool_calls fields'. Card: roles system/user/assistant/tool. Merges claims 0, 1, 11, 12. CAVEAT (does not refute structure): traces are synthetically distilled by Minimax-M2.5 + Qwen3.5-122B against ~20k real PRs — not organic production logs. Practical caveat from MarkTechPost tutorial: 'normalized' schema is imperfect — trajectory entries may be dicts OR serialized JSON strings needing defensive parsing.

Sources: https://huggingface.co/datasets/nvidia/Open-SWE-Traces, https://arxiv.org/html/2606.16038

### [high] nebius/SWE-agent-trajectories: 80,036 SWE-agent trajectories (13,389 successful / 66,647 unsuccessful) over GitHub issues, CC-BY-4.0 (plus Llama 3.1 License compliance for model outputs), 1.11 GB Parquet — each row a JSON list alternating model reasoning/actions (role: ai) with environment observations (role: user), with separate trajectory / generated_patch / eval_logs fields.

Card: 80,036 total, 13,389 successful, 66,647 unsuccessful (arithmetic consistent); trajectory field 'A JSON list with the logged trajectory consisting of a list of model reasoning and actions (under the role: ai) and observations from the environment (under the role: user)'; 'licensed under the Creative Commons Attribution 4.0 license' with additional Llama 3.1 License compliance for model outputs, plus per-repo license respect. Backs Nebius's 40.6% SWE-bench Verified result. Merges claims 13, 14, 15. NOTE: SWE-agent-native JSON needs a schema-adapter for ingestion.

Sources: https://huggingface.co/datasets/nebius/SWE-agent-trajectories

### [high] nebius/SWE-rebench-openhands-trajectories: 67,074 OpenHands trajectories (32,161 successful, ~48%) across 1,823 Python repos, CC-BY-4.0, ~2.08 GB Parquet, averaging 64.3 turns (max 100) — deep enough to bury answer spans in earlier turns. Full conversation history (system/assistant/user/tool with tool_calls + tool_call_id), model_patch as a separate field. A strong first-ingest candidate for the buried-evidence golden.

Card + Nebius blog (2025-12-23) both: 'Total trajectories: 67,074', 'Successful trajectories: 32,161', 'Average turn count: 64.3', 'Maximum turn count: 100'; generated with OpenHands v0.54.0 + Qwen3-Coder-480B function calling. Trajectory field = per-role conversation with assistant tool_calls and tool messages carrying name/tool_call_id — definitional multi-turn tool-use signature. Merges claims 2, 3, 4. QA-HYGIENE CAVEAT: 52% are failed attempts and independent research (arxiv 2511.00197, 2604.02547) shows failed trajectories run +14% to +112% longer via looping/step-repetition, so the 64.3 average is partly padded — dedupe/filter looping turns when building QA.

Sources: https://huggingface.co/datasets/nebius/SWE-rebench-openhands-trajectories, https://nebius.com/blog/posts/openhands-trajectories-with-qwen3-coder-480b

### [high] SWE-bench/SWE-smith-trajectories is a small, clean, MIT-licensed set of 5,017 real SWE-agent trajectories (SWE-agent + Claude 3.7 Sonnet on SWE-smith task instances), each a `messages` OpenAI-format conversation (7.95k-664k chars) alongside instance_id/resolved/model/traj_id/patch, in three tool-format splits (tool/xml/ticks). Best low-friction first ingest for schema-plumbing validation.

Card frontmatter 'license: mit'; README '5017 trajectories we fine-tuned Qwen 2.5 Coder Instruct on, leading to SWE-agent-LM-32B'; fields messages/instance_id/resolved/model/traj_id/patch; splits tool=24,100, xml=26,076, ticks=25,826 (naive row count ~76k triples the 5,017 trajectory count across formats). 'Trajectories were generated by running SWE-agent + Claude 3.7 Sonnet on task instances from the SWE-smith dataset.' Merges claims 5, 6, 7. Note: underlying SWE-smith bug TASKS are synthetic but the trajectories over them are real agent runs.

Sources: https://huggingface.co/datasets/SWE-bench/SWE-smith-trajectories, https://arxiv.org/abs/2504.21798

### [high] SWE-Gym releases downloadable multi-turn trajectory sets on HuggingFace — OpenHands-Sampled-Trajectories (~6,055 rows), OpenHands-Verifier-Trajectories (~5.27k), OpenHands-SFT-Trajectories (491, MIT), and MoatlessTools sets — each with a `messages` field (7-101 messages), `tools`, `resolved`, `test_result`. Plus a large task substrate (~2,438 verified Python instances, 230-instance Lite, 64.7K-row raw split) with executable environments to re-generate rollouts if needed.

HF org page confirms row counts exactly (OpenHands-Sampled 6,055/301MB, Verifier 5.27k, SFT 491/MIT, MoatlessTools-Agent-Verifier-Train-Data 2.16k); schema has messages/tools/resolved/test_result. ICML 2025 paper (2412.21139): '2,438 real-world Python task instances, each comprising a codebase with an executable runtime environment, unit tests'; SWE-Gym-Lite 230 (HF card), SWE-Gym-Raw 64,689 across 358 repos. Merges claims 8, 9. CAVEAT: SWE-Gym-Raw is 'raw and unvalidated' (most instances lack configured Python env); only the 2,438-instance main split is executable/verified.

Sources: https://github.com/SWE-Gym/SWE-Gym, https://huggingface.co/SWE-Gym, https://arxiv.org/abs/2412.21139

### [high] SWE-ContextBench is the strongest 2026 template for the buried-evidence golden: it builds a continuity corpus by running Claude Code (Claude Sonnet 4.5) on 1,100 experience tasks and storing each FULL multi-turn trajectory (tool calls normalized to inline text, file navigation, reasoning; avg 25,633.7 tokens, vs 217.1-token summaries) as a retrievable context pool — the assistant+toolResult transcript format the pipeline ingests.

PDF-verified (pdftotext): 'we run a base coding agent and record its full trajectory, including tool calls, file navigation, and intermediate reasoning steps. These trajectories are stored as past contexts and form a context pool that agents can later retrieve'; 'we run Claude Code (Claude Sonnet 4.5) on the 1,100 experience tasks'; 'summary files contain an average of 217.1 tokens, while the full context trajectories average 25,633.7 tokens'; 'tool calls normalized to inline text'. Claim 16, 3-0. (Note: a same-family but DISTINCT benchmark 'ContextBench', arXiv 2602.05892, is easily conflated — not this source.)

Sources: https://arxiv.org/abs/2602.08316

### [high] SWE-ContextBench's buried-evidence golden is built by human-verified GitHub issue/PR dependency mining, NOT LLM span-located QA: 319 interdependent instances found across 1,100 base instances via six workflow types (multi-issue, PR-to-issue, PR-only, issue-to-issue, issue-to-PR, multi-reference), + 57 via recursive expansion = 376 related tasks, 1,476 total across 51 repos / 9 languages — the answer to a related task is buried in an earlier experience-task trajectory. Grading is span-agnostic and test-based (base_commit + problem_statement + test_patch + solution_patch, scored on FAIL_TO_PASS avg 4.8 / PASS_TO_PASS avg 211.5).

'Across the 1,100 base instances, we identify 319 such interdependent task instances'; recursion 'yields 57 additional'; 319+57=376; 'SWE-ContextBench contains 1,476 tasks from 51 unique GitHub repositories across 9 programming languages' (1,100+376). 'Each candidate task instance is manually verified' — dependency mining, not LLM span-QA. Grading: 'FAIL_TO_PASS set includes tests that fail... but pass in results_after'; 'PASS_TO_PASS... ensuring no regressions'; 'each of the 376 related tasks contains 4.8 FAIL_TO_PASS tests and 211.5 PASS_TO_PASS tests'. Merges claims 17, 18. Version note: v1 covered only the Lite subset (300/99), so naive fetches report smaller numbers; these are v2 full-benchmark aggregates. This is a concrete alternative to LLM span-containment grading, but it grades whether the agent's patch resolves the task, not retrieval correctness per se.

Sources: https://arxiv.org/abs/2602.08316

### [high] LongMemEval supplies the complementary needle-in-haystack recipe that directly matches the pipeline's LLM span-located QA approach: evidence sessions are randomly inserted into the middle of sampled unrelated distractor chat sessions with plausible assigned timestamps; distractors come from simulated self-chats (facts from user backgrounds, non-conflicting attributes) plus public data (ShareGPT, UltraChat).

arXiv 2410.10813 (ICLR 2025): 'we sample a number of unrelated user-AI chat sessions, randomly insert the evidence sessions in the middle, and assign a plausible timestamp to each session'; 'We draw the irrelevant sessions from two sources: (1) self-chat sessions simulated using facts extracted from the user backgrounds... and (2) publicly released user-AI style chat data including ShareGPT and UltraChat.' Official repo ships sample_haystack_and_timestamp.py. Claim 22, 3-0. Refinement: insertion is random among shuffled distractors; when evidence needs temporal order the paper re-swaps after shuffling.

Sources: https://arxiv.org/abs/2410.10813

### [medium] SWE-chat (SALT-NLP/SWE-chat, arXiv 2604.20779) is a real-user coding-agent session dataset with full tool-call trajectories (user_prompt/assistant_response/assistant_thinking/tool_use/tool_result/tool_name/command/file_path per turn, per-session JSONL, commits.parquet with human/agent-attributed diffs), odc-by license — a rare source of ORGANIC (in-the-wild human-triggered) traces rather than synthetic rollouts.

arXiv 2604.20779 'SWE-chat: Coding Agent Interactions From Real Users in the Wild': 'Each session consists of alternating user prompts and agent responses with tool calls (file reads, edits, shell commands) and text output'; 'complete interaction traces between humans and AI coding agents, with full tool-call trajectories and code diffs'. HF schema (conversations.parquet 2.69M turn rows) confirmed. Claim 10 (format) confirmed 3-0. MEDIUM because the companion volume claim (6,000 sessions / 63k prompts / 355k tool calls / 2.7M events) was REFUTED (vote 1-2) on a version/count discrepancy (HF shows ~5,851 sessions), and license is odc-by (attribution/share — check share-alike-style terms before commercial ingest). The multi-turn tool-use FORMAT is solid; exact volume figures are uncertain.

Sources: https://arxiv.org/html/2604.20779v1, https://huggingface.co/datasets/SALT-NLP/SWE-chat
